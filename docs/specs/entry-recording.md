## 記録の登録

- ラベルはユーザーの自由入力（必須）
- 通貨は日本円のみ
- 金額は0円以上（負の値は不可）
- 登録者は常に本人（パートナーによる代理登録はなし）
- レシート画像は最大2枚まで添付可能。削除・追加は可能（差し替えは削除+追加で対応）
- 重複登録チェックは将来検討

## 記録・精算の修正・取り消し（バージョン管理方式）

記録や精算の変更履歴をバージョン管理で保持する。直接編集・削除は行わず、新しいバージョンのレコードを作成する。

### 基本概念

- 修正すると新しいレコードが作成される。金額・ラベル・メモなど**修正後のフルスナップショット**を保存する（差分ではない）
- 取り消すと `cancelled = true` の新バージョンが作成される
- 各バージョンは `original_id` で初版と紐付く（初版は `original_id = 自身の id`）
- `latest` フラグにより最新バージョンを即座に特定可能
- バージョン順は `created_at` で判定（`version` カラムは不要。正規化の観点から排除）
- 金額は全バージョンで常に 0 以上（正の値）

### 修正の動作

1. 元の記録の `latest` を `false` に更新
2. 修正後の値で新レコードを作成（`latest = true`, `original_id = 元の初版ID`）
3. 例: 10,000円の立替を9,000円に修正 → 新レコード `{ amount: 9000, latest: true }`
4. タイムラインには「¥9,000 修正」として表示。元の記録は「修正済み」ステータスに

### 取り消しの動作

1. 元の記録の `latest` を `false` に更新
2. `cancelled = true` の新レコードを作成（`latest = true`, `original_id = 元の初版ID`）
3. `amount` には取消直前の金額を保持（表示用: 「¥9,000 取消済み」）
4. タイムラインには「取消済み」として表示

### 取消の復元

1. 取消済み記録の `latest` を `false` に更新
2. `cancelled = false` の新レコードを作成（`latest = true`, `original_id = 元の初版ID`）
3. 取消前の金額・ラベル・メモが復元される
4. タイムラインには通常の記録として再表示される

### 残高計算

```
latest = true AND cancelled = false のレコードのみ対象
残高 = SUM(CASE WHEN category = 'advance' THEN amount ELSE -amount END)
```

### 管理モード以外（ソロ・共有）の記録

- 修正・取り消しは記録の詳細画面から行う
- タイムラインに変更履歴が残り、自分とパートナーの両者にとって変更内容が明確になる
- 管理モード時、修正・取り消しレコードにも再度パートナーの承認が必要

### 管理モードで承認待ちの記録

- 当該レコードを直接編集・削除する
- 新バージョンは作成しない

### 管理モードで差し戻された記録

- 承認済みの記録と同様に、新バージョンを作成する
- 修正・取り消しレコードにも再度パートナーの承認が必要
