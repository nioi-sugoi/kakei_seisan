## パートナー招待機能

### 概要
アプリを利用する2人（パートナー）を紐付けるための機能。
関係は1対1で、1人のユーザーが持てるパートナーは1人だけ。

### 招待の流れ
1. User A がアプリ内でパートナーのメールアドレスを入力して「招待」を実行する
2. User B がアプリにログインすると、自分宛の招待が表示される
3. User B が承認すると、2人のパートナー関係が成立する

### ルール
- 招待には有効期限がある（48時間）
- すでにパートナーがいるユーザーは招待の送信・承認ができない
- 招待先のユーザーがアプリ未登録の場合、招待は期限切れまで pending のまま保持される
- 招待先のユーザーが既にパートナーを持っている場合、招待を送信できない
- 同一ユーザーが有効期限内の招待を複数送信することはできない
- 招待者は pending 状態の招待を解除（取り消し）できる。解除された招待はDBから削除される
- パートナー解除機能は現時点では設けない
- 招待を送信した側（招待者）はパートナー関係において管理モードの変更権限を持つ。パートナー関係テーブルに招待者を記録する

### 対象外
- 未登録ユーザーへのアプリインストール案内メール送信（将来対応）
