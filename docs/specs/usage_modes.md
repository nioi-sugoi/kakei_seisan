## 利用モード

- **ソロ**: パートナー未連携。自分の記録・精算のみを管理する
- **共有**: パートナー連携済み、管理モードなし。互いの記録・精算を閲覧できる
- **管理**: パートナー連携済み、管理モードあり。記録・精算にパートナーの承認が必要になる

### モードの設定ルール
- 各ユーザーが異なるモードで運用可能（例: User Aは管理モード、User Bは共有モード）
- パートナー連携直後は両者とも共有モードで開始する
- モードの変更は**合意制**。一方が変更を提案し、もう一方が承諾して初めて反映される
  - 自分のモードを変更する場合も、相手のモードを変更する場合も、相手の承諾が必要
  - 管理→共有への変更も合意必須（承認待ちレコードの自動承認が発生するため）

### モード変更の合意プロセス
1. 提案者がモード変更を提案する（「User X のモードを Y にする」）
2. 相手（非提案者）が承諾または拒否する
3. 承諾された場合のみモードが変更される

- 提案は1ペアにつき1件まで。新しい提案を作ると既存の未承諾提案は自動キャンセルされる
- 提案者は承諾前であれば提案を撤回できる

### モード切り替え時の挙動
- **共有→管理**: 切り替え以降の新規レコードのみ承認が必要。既存レコードには影響なし
- **管理→共有**: 対象ユーザーの承認待ちレコードはすべて自動承認扱いとなり、残高に反映される。差し戻しレコードはそのまま残る（残高には反映されない）

#### 管理→共有 合意画面での明示事項
合意画面では以下の処理内容を提案者・承諾者の双方に表示する:
- 承認待ちの記録 N件 → 承認済みに（残高に反映）
- 承認待ちの取消 N件 → 取消確定
- 差し戻しの記録 N件 → そのまま残る
